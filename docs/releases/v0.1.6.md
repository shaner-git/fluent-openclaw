# Fluent for OpenClaw v0.1.6

`fluent-openclaw` `v0.1.6` is the current canonical public beta package for OpenClaw.

- Supported deployment tracks: `cloud` and `oss`
- Minimum Fluent MCP contract: `2026-04-20.fluent-core-v1.37`
- OpenClaw plugin API floor: `2026.3.24-beta.2`

## Install

```bash
openclaw plugins install fluent-openclaw
```

The Fluent OSS repo also carries an embedded OpenClaw helper bundle exported from `fluent-mcp/openclaw-plugin/fluent`, but it is not the same package artifact as the published `fluent-openclaw` release.

## Hosted Setup

1. Install `fluent-openclaw`.
2. Run `openclaw fluent auth login`.
3. Finish sign-in in the browser window opened by OpenClaw.
4. Run `openclaw fluent mcp cloud`.
5. Run `openclaw fluent doctor`.

## Self-Hosted Setup

1. Start Fluent OSS and expose it at `http://127.0.0.1:8788`.
2. Provide an OSS token with `FLUENT_OSS_TOKEN`, an existing local Fluent OSS bootstrap, or `--token`.
3. Run `openclaw fluent mcp oss --base-url http://127.0.0.1:8788`.
4. Run `openclaw fluent doctor oss --base-url http://127.0.0.1:8788`.

## Known Limitations

- `fluent-openclaw` is public beta, not GA.
- Hosted Fluent rollout and plugin-managed hosted auth are still beta.
- OpenClaw uses the canonical Fluent MCP tools and resources, but it does not render Fluent widget surfaces or output templates.
- OSS setup still requires a bearer token.
- Browser execution helpers are intentionally omitted from this package.
- Retailer automation helpers are intentionally omitted from this package.
- Unsafe local browser automation and retailer checkout runners are not required for standard installs and are not shipped here.

## Intentional Scope

This package ships Fluent skills, hosted auth helpers, and MCP setup flows for OpenClaw. Browser and retailer execution helpers are intentionally omitted from the public package.

The embedded `fluent-oss/openclaw-plugin/fluent` bundle is a separate OSS helper package with its own metadata and versioning. It is not generated from this package line, and this package line is not generated from it.
