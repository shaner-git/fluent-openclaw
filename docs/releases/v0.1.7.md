# Fluent for OpenClaw v0.1.7

`fluent-openclaw` `v0.1.7` is the current canonical public beta package for OpenClaw.

- Setup command labels: `cloud` and `oss`
- Minimum Fluent MCP contract: `2026-06-01.fluent-core-v1.85`
- OpenClaw plugin API floor: `2026.3.24-beta.2`

## What Changed Since v0.1.6

- Bumped the package and OpenClaw plugin metadata to `0.1.7` for a fresh ClawHub publish.
- Raised both the minimum compatible and current reference Fluent MCP contract versions to `2026-06-01.fluent-core-v1.85`.
- Refreshed the packaged Fluent skills from `fluent-mcp/openclaw-plugin/fluent`, including current Meals planning, Style closet, Budgets, and visual-boundary guidance.
- Updated hosted auth to request the canonical `/mcp` OAuth resource and kept the public hosted MCP endpoint at `https://mcp.meetfluent.app/mcp`.

## Install

```bash
openclaw plugins install fluent-openclaw
```

The Fluent open-source runtime repo also carries an embedded OpenClaw helper bundle exported from `fluent-mcp/openclaw-plugin/fluent`, but it is not the same package artifact as the published `fluent-openclaw` release.

## Early-Access Setup

1. Install `fluent-openclaw`.
2. Run `openclaw fluent auth login`.
3. Finish sign-in in the browser window opened by OpenClaw.
4. Run `openclaw fluent mcp --track cloud`.
5. Run `openclaw fluent doctor`.

## Run Fluent Yourself

1. Start Fluent open-source runtime and expose it at `http://127.0.0.1:8788`.
2. Provide an OSS token with `FLUENT_OSS_TOKEN`, an existing local Fluent open-source runtime bootstrap, or `--token`.
3. Run `openclaw fluent mcp --track oss --base-url http://127.0.0.1:8788`.
4. Run `openclaw fluent doctor --track oss --base-url http://127.0.0.1:8788`.

## Known Limitations

- `fluent-openclaw` is public beta, not GA.
- Fluent is invite-based early access and currently free, and plugin-managed hosted auth is still beta.
- OpenClaw uses Fluent's canonical public vNext tools and promoted MCP resources, but OpenClaw should default to text/data unless the host explicitly proves widget support.
- Open-source runtime setup still requires a bearer token.
- Browser execution helpers are intentionally omitted from this package.
- Retailer automation helpers are intentionally omitted from this package.
- Unsafe local browser automation and retailer checkout runners are not required for standard installs and are not shipped here.

## Intentional Scope

This package ships Fluent skills, hosted auth helpers, and MCP setup flows for OpenClaw. Browser and retailer execution helpers are intentionally omitted from the public package.

The embedded `fluent-oss/openclaw-plugin/fluent` bundle is a separate OSS helper package with its own metadata and versioning. It is not generated from this package line, and this package line is not generated from it.
